# Roadmap
## Telegram Booking Platform

---

Дорожная карта разработки платформы онлайн-записи. Основана на [PRD](prd.md), [Architecture](architecture.md) и [Project Overview](project.md).

---

# 1. Обзор

## Цель MVP

Доставить работающую платформу, в которой:

- бизнес настраивает компанию, услуги, сотрудников и расписание через Web Admin
- клиенты записываются на услуги через Telegram Mini App
- мастер и клиент получают уведомления через Telegram Bot

## Ключевые зависимости

- Backend API — основа для Admin, Mini App и Bot
- База данных — схемы и миграции
- Redis — очереди уведомлений и кеширование

# 2. Фазы разработки

## Фаза 0: Инфраструктура и основа

**Цель:** Подготовить монорепозиторий, общую конфигурацию и базу данных.

| Задача | Описание |
|--------|----------|
| Monorepo | pnpm workspaces + Turborepo |
| Shared packages | `prisma`, `ui`, `eslint-config`, `typescript-config` |
| Database | PostgreSQL + Prisma, миграции |
| Redis | Настройка кеша и очередей (ConfigModule, RedisModule) |
| MinIO | Объектное хранилище (StorageModule) |
| Docker | docker-compose (PostgreSQL, Redis, MinIO), Dockerfile в apps |
| Backend shell | NestJS, ConfigModule, Redis, MinIO, health check |

**Сущности для реализации:** AdminUser, Business, ServiceCategory, Service, Employee, Schedule, Booking, Client, TimeSlot

# 5. После MVP

| Задача | Описание |
|--------|----------|
| Подтверждение email | Верификация email при регистрации (поле `emailVerifiedAt` уже есть в AdminUser). Flow: письмо со ссылкой → подтверждение → доступ к полному функционалу |
| CDN для файлов | Подключение CDN перед MinIO для раздачи статики. Меняется только `FILE_STORAGE_URL` в конфиге |
| OAuth | Вход через Google и др. (архитектура AuthStrategy уже предусматривает расширение) |
| Роль ADMIN | Расширение BusinessMember: роль ADMIN для администраторов без права удаления бизнеса |
| Несколько бизнесов | Один пользователь — несколько бизнесов. Снятие ограничения MVP, UI для переключения |
| Slug для бизнеса | URL-friendly идентификатор для deep link в Mini App |
| Gap strategy: race condition при ребалансе | Транзакции или блокировки при пересчёте позиций при параллельных reorder (Feature 1.3) |
