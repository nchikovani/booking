# Roadmap
## Telegram Booking Platform

---

Дорожная карта разработки платформы онлайн-записи. Основана на [PRD](prd.md), [Architecture](architecture.md) и [Project Overview](project.md).

---

# 1. Обзор

## Цель MVP

Доставить работающую платформу, в которой:

- бизнес настраивает компанию, услуги, сотрудников и расписание через Web Admin
- клиенты записываются на услуги через Telegram Mini App
- мастер и клиент получают уведомления через Telegram Bot

## Ключевые зависимости

- Backend API — основа для Admin, Mini App и Bot
- База данных — схемы и миграции
- Redis — очереди уведомлений и кеширование

---

# 2. Фазы разработки

## Фаза 0: Инфраструктура и основа

**Цель:** Подготовить монорепозиторий, общую конфигурацию и базу данных.

| Задача | Описание |
|--------|----------|
| Monorepo | pnpm workspaces + Turborepo |
| Shared packages | `types`, `validation`, `ui`, `config` |
| Database | PostgreSQL + Prisma, схема и миграции |
| Redis | Настройка кеша и очередей |
| Docker | Конфигурация для локальной разработки |
| Backend shell | NestJS, базовая структура API |

**Сущности для реализации:** User, Business, ServiceCategory, Service, Employee, Schedule, Booking, Client, TimeSlot

---

## Фаза 1: Backend — ядро API

**Цель:** Реализовать Admin API и Client API (базовая логика).

### Admin API (`/api/v1/admin`)

| Этап | Задачи |
|------|--------|
| 1.1 | Аутентификация, middleware |
| 1.2 | Бизнес: CRUD, настройки компании |
| 1.3 | Категории услуг, услуги (CRUD) |
| 1.4 | Сотрудники (CRUD, связь с услугами) |
| 1.5 | Расписание: график работы, перерывы |
| 1.6 | Записи: список, создание вручную, перенос, отмена |
| 1.7 | Клиенты: список, просмотр истории |

### Client Mini App API (`/api/v1/app`)

| Этап | Задачи |
|------|--------|
| 1.8 | Аутентификация (Telegram initData), middleware |
| 1.9 | Публичная информация о компании |
| 1.10 | Список услуг и сотрудников |
| 1.11 | Алгоритм слотов: расчёт доступных дат и времени |
| 1.12 | Создание записи |
| 1.13 | Перенос и отмена записи |
| 1.14 | Список своих записей (по Telegram user) |

---

## Фаза 2: Web Admin

**Цель:** Веб-панель для владельца бизнеса.

| Этап | Страница / раздел | Функции |
|------|-------------------|---------|
| 2.1 | Onboarding & Auth | Вход через Telegram |
| 2.2 | Настройки компании | Название, логотип, описание, контакты, адрес |
| 2.3 | Услуги | Категории, CRUD услуг |
| 2.4 | Сотрудники | Список, создание, редактирование, привязка услуг |
| 2.5 | График работы | Рабочие дни, часы, перерывы |
| 2.6 | Календарь записей | Просмотр, перенос, отмена, создание вручную |
| 2.7 | База клиентов | Список, профиль, история записей |
| 2.8 | Панель управления | Дашборд, сводка |

**Стек:** Next.js, React, TypeScript, Tailwind, Zustand, React Query, FSD

---

## Фаза 3: Telegram Mini App

**Цель:** Клиентский флоу записи в Telegram.

| Этап | Страница | Функции |
|------|----------|---------|
| 3.1 | Настройка Mini App | @tma.js/sdk-react, initData |
| 3.2 | Главная | Навигация, приветствие |
| 3.3 | Информация о компании | Название, лого, описание, фото, контакты |
| 3.4 | Выбор специалиста | Список сотрудников |
| 3.5 | Выбор услуги | Список услуг |
| 3.6 | Выбор даты и времени | Календарь, слоты |
| 3.7 | Подтверждение записи | Итог, подтверждение |
| 3.8 | Мои записи | Список, детали, перенос, отмена |

**Стек:** Next.js, React, TypeScript, Tailwind, Zustand, React Query, FSD, @tma.js/sdk-react

---

## Фаза 4: Telegram Bot

**Цель:** Уведомления и работа с расписанием.

### Bot API (`/api/v1/bot`)

| Этап | Задача |
|------|--------|
| 4.1 | Webhook для событий Telegram |
| 4.2 | Генерация deep link для Mini App |
| 4.3 | Очередь уведомлений (Redis) |

### Функции бота

| Этап | Функция | Описание |
|------|---------|----------|
| 4.4 | Уведомление мастеру | При новой записи: сообщение + кнопки |
| 4.5 | Напоминания клиенту | За 24 ч и 2 ч до визита |
| 4.6 | Команды мастера | `/today`, `/tomorrow`, `/week` |
| 4.7 | Inline-кнопки | «Открыть запись», «Отменить запись» |

---

## Фаза 5: Безопасность и надёжность

| Задача | Описание |
|--------|----------|
| 5.1 | Проверка подписи Telegram initData | Валидация на backend |
| 5.2 | Валидация входных данных | DTO, схемы |
| 5.3 | Проверка прав доступа | Роли, контекст бизнеса |
| 5.4 | Защита от дублей | Блокировка при одновременной записи |
| 5.5 | Rate limiting | На создание записей |
