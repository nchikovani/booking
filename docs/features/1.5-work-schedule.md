## 1. Обзор

Реализация графика работы сотрудников в Admin API: шаблоны расписания и управление графиком сотрудника через API сотрудников (Feature 1.4). График задаётся по дням недели с рабочими часами и перерывами. Сотрудник возвращается вместе с графиком; при создании и редактировании сотрудника можно задать график (шаблон или индивидуальный). При назначении шаблона индивидуальный график автоматически удаляется. Данные используются в Feature 2.4 (алгоритм слотов) для расчёта доступного времени записи.

### 1.1. Область реализации

| Область             | Применимо | Примечание                                                                                                                               |

| Сущность                     | Назначение                                                           |
| ---------------------------- | -------------------------------------------------------------------- |

## 4. Технический дизайн

### 4.1. Модель владения

Шаблоны принадлежат бизнесу (`businessId`). График сотрудника привязан к Employee. Доступ контролируется через `requireBusinessMember(adminUserId, businessId)`.

### 4.2. Дни недели

| Значение | День        |
| -------- | ----------- |
| 0        | Воскресенье |
| 1        | Понедельник |
| 2        | Вторник     |
| 3        | Среда       |
| 4        | Четверг     |
| 5        | Пятница     |
| 6        | Суббота     |

Соответствует JavaScript `Date.getDay()` и PostgreSQL `EXTRACT(DOW FROM date)`.

### 4.3. Хранение времени

- В БД: `Int` — минуты от полуночи (0–1439), кратно 5.
- В API: строка `"HH:mm"` (например, `"09:00"`, `"17:30"`).
- Валидация: парсинг `HH:mm`, проверка `minutes % 5 === 0`.

### 4.4. Логика назначения графика

- **EmployeeSchedule** — связь 1:1 с Employee. Управление через Employee API (create/update).
- **Шаблон:** `scheduleTemplateId` задан, `EmployeeScheduleDay` не используется. При назначении шаблона индивидуальный график (EmployeeScheduleDay, EmployeeScheduleDayBreak) автоматически удаляется.
- **Индивидуальный:** `scheduleTemplateId = null`, используются `EmployeeScheduleDay` и `EmployeeScheduleDayBreak`.
- **Без графика:** `schedule` не передан или передан `null` — EmployeeSchedule отсутствует или удаляется. При create: отсутствие schedule = сотрудник без графика. При update: отсутствие schedule = не менять график.
- **Индивидуальный с пустым days:** `schedule: { days: [] }` допустимо — сотрудник без рабочих дней.

## 5. Схема базы данных

### 5.1. Таблица `ScheduleTemplate`

```prisma
model ScheduleTemplate {
  id         String   @id @default(uuid())
  businessId String   @map("business_id")
  name       String
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt

  business         Business             @relation(fields: [businessId], references: [id], onDelete: Cascade)
  days             ScheduleTemplateDay[]
  employeeSchedules EmployeeSchedule[]

  @@index([businessId])
  @@map("schedule_templates")
}
```

### 5.2. Таблица `ScheduleTemplateDay`

```prisma
model ScheduleTemplateDay {
  id                 String   @id @default(uuid())
  scheduleTemplateId String   @map("schedule_template_id")
  dayOfWeek          Int      @map("day_of_week")  // 0-6: вс-сб
  startTimeMinutes    Int      @map("start_time_minutes")  // минуты от полуночи, кратно 5
  endTimeMinutes      Int      @map("end_time_minutes")    // минуты от полуночи, кратно 5
  createdAt          DateTime @default(now())
  updatedAt          DateTime @updatedAt

  scheduleTemplate ScheduleTemplate @relation(fields: [scheduleTemplateId], references: [id], onDelete: Cascade)
  breaks           ScheduleTemplateDayBreak[]

  @@unique([scheduleTemplateId, dayOfWeek])
  @@index([scheduleTemplateId])
  @@map("schedule_template_days")
}
```

### 5.3. Таблица `ScheduleTemplateDayBreak`

```prisma
model ScheduleTemplateDayBreak {
  id                    String   @id @default(uuid())
  scheduleTemplateDayId  String   @map("schedule_template_day_id")
  startTimeMinutes       Int      @map("start_time_minutes")
  endTimeMinutes         Int      @map("end_time_minutes")
  createdAt              DateTime @default(now())

  scheduleTemplateDay ScheduleTemplateDay @relation(fields: [scheduleTemplateDayId], references: [id], onDelete: Cascade)

  @@index([scheduleTemplateDayId])
  @@map("schedule_template_day_breaks")
}
```

### 5.4. Таблица `EmployeeSchedule`

```prisma
model EmployeeSchedule {
  id                 String    @id @default(uuid())
  employeeId         String    @unique @map("employee_id")
  scheduleTemplateId String?   @map("schedule_template_id")  // null = индивидуальный график
  createdAt          DateTime  @default(now())
  updatedAt          DateTime  @updatedAt

  employee         Employee          @relation(fields: [employeeId], references: [id], onDelete: Cascade)
  scheduleTemplate ScheduleTemplate? @relation(fields: [scheduleTemplateId], references: [id], onDelete: SetNull)
  days             EmployeeScheduleDay[]

  @@index([employeeId])
  @@index([scheduleTemplateId])
  @@map("employee_schedules")
}
```

## 6. API эндпоинты

Base path: `/api/v1/admin/businesses/:businessId`

| Метод  | Путь                      | Описание           | Доступ                     |
| ------ | ------------------------- | ------------------ | -------------------------- |
| GET    | `/schedule-templates`     | Список шаблонов    | Access token, член бизнеса |
| POST   | `/schedule-templates`     | Создание шаблона   | Access token, член бизнеса |
| GET    | `/schedule-templates/:id` | Получение шаблона  | Access token, член бизнеса |
| PATCH  | `/schedule-templates/:id` | Обновление шаблона | Access token, член бизнеса |
| DELETE | `/schedule-templates/:id` | Удаление шаблона   | Access token, член бизнеса |

График сотрудника — см. [Feature 1.4](1.4-employees.md): GET/POST/PATCH `/employees` возвращают/принимают поле `schedule`.

### 6.1. Список шаблонов

**GET** `/api/v1/admin/businesses/:businessId/schedule-templates`

**Response 200:**

```json
{
  "status": "success",
  "data": [
    {
      "id": "uuid",
      "name": "Стандартный график",
      "days": [
        {
          "dayOfWeek": 1,
          "startTime": "09:00",
          "endTime": "18:00",
          "breaks": [{ "startTime": "13:00", "endTime": "14:00" }]
        }
      ],
      "createdAt": "2024-01-15T10:00:00.000Z"
    }
  ]
}
```

**Errors:** `401 UNAUTHORIZED` | `404 NOT_FOUND`

### 6.2. Создание шаблона

**POST** `/api/v1/admin/businesses/:businessId/schedule-templates`

**Request body:**

```json
{
  "name": "Стандартный график",
  "days": [
    {
      "dayOfWeek": 1,
      "startTime": "09:00",
      "endTime": "18:00",
      "breaks": [{ "startTime": "13:00", "endTime": "14:00" }]
    },
    {
      "dayOfWeek": 2,
      "startTime": "09:00",
      "endTime": "18:00",
      "breaks": []
    }
  ]
}
```

- `dayOfWeek`: 0–6
- `startTime`, `endTime`: `"HH:mm"`, кратно 5 минут
- `breaks`: массив `{ startTime, endTime }`, время кратно 5
- `startTime` < `endTime` для дня и для каждого перерыва
- Перерывы внутри рабочего дня: `startTime ≤ breakStart < breakEnd ≤ endTime`

**Response 201:**

```json
{
  "status": "success",
  "data": {
    "id": "uuid",
    "name": "Стандартный график",
    "days": [
      /* ... */
    ],
    "createdAt": "2024-01-15T10:00:00.000Z"
  }
}
```

**Errors:** `400 VALIDATION_FAILED` | `401 UNAUTHORIZED` | `404 NOT_FOUND`

### 6.3. Получение шаблона

**GET** `/api/v1/admin/businesses/:businessId/schedule-templates/:id`

**Response 200:** `{ "status": "success", "data": { /* шаблон с days */ } }`

**Errors:** `401 UNAUTHORIZED` | `404 NOT_FOUND`

### 6.4. Обновление шаблона

**PATCH** `/api/v1/admin/businesses/:businessId/schedule-templates/:id`

Частичное обновление. `name` и `days` опциональны. `days` при передаче полностью заменяет текущие дни.

**Request body:**

```json
{
  "name": "Обновлённый график",
  "days": [
    /* полная замена */
  ]
}
```

**Response 200:** `{ "status": "success", "data": { /* обновлённый шаблон */ } }`

**Errors:** `400 VALIDATION_FAILED` | `401 UNAUTHORIZED` | `404 NOT_FOUND`

### 6.5. Удаление шаблона

**DELETE** `/api/v1/admin/businesses/:businessId/schedule-templates/:id`

- Перед удалением шаблона: удалить EmployeeSchedule у сотрудников с scheduleTemplateId = id удаляемого шаблона (у них days пустой; после — `schedule: null`)
- onDelete: SetNull — для оставшихся связей (если не удалили заранее)
- Удаление шаблона и связанных дней/перерывов

**Response 200:** `{ "status": "success", "data": null }`

**Errors:** `401 UNAUTHORIZED` | `404 NOT_FOUND`

## 8. DTO и валидация

### 8.1. ScheduleTemplateDayDto (вложенный)

| Поле      | Тип      | Required | Валидация                                                                                 |
| --------- | -------- | -------- | ----------------------------------------------------------------------------------------- |
| dayOfWeek | number   | ☑        | 0–6                                                                                       |
| startTime | string   | ☑        | HH:mm, кратно 5                                                                           |
| endTime   | string   | ☑        | HH:mm, кратно 5, строго > startTime (равенство — 400)                                     |
| breaks    | object[] | ☐        | массив BreakDto, max 10; перерывы не пересекаются; каждый внутри [startTime, endTime] дня |

### 8.2. ScheduleTemplateBreakDto (вложенный)

| Поле      | Тип    | Required | Валидация                                             |
| --------- | ------ | -------- | ----------------------------------------------------- |
| startTime | string | ☑        | HH:mm, кратно 5                                       |
| endTime   | string | ☑        | HH:mm, кратно 5, строго > startTime (равенство — 400) |

### 8.3. CreateScheduleTemplateDto

| Поле | Тип                      | Required | Валидация                          |
| ---- | ------------------------ | -------- | ---------------------------------- |
| name | string                   | ☑        | min 1, max 200                     |
| days | ScheduleTemplateDayDto[] | ☑        | min 1, max 7, уникальные dayOfWeek |

### 8.4. UpdateScheduleTemplateDto

| Поле | Тип                      | Required | Валидация                                                                             |
| ---- | ------------------------ | -------- | ------------------------------------------------------------------------------------- |
| name | string                   | ☐        | min 1, max 200                                                                        |
| days | ScheduleTemplateDayDto[] | ☐        | min 1, max 7, уникальные dayOfWeek; полная замена; `days: []` — 400 VALIDATION_FAILED |

### 8.5. ScheduleDto (вложенный в CreateEmployeeDto / UpdateEmployeeDto)

См. [Feature 1.4](1.4-employees.md), раздел 8. Один из вариантов:

- `scheduleTemplateId`: string (UUID)
- `days`: ScheduleTemplateDayDto[] (массив дней)

Валидация: ровно одно из полей задано, либо `schedule: null` для удаления графика. Пустой объект `{}` без полей — 400 VALIDATION_FAILED. Одновременная передача `scheduleTemplateId` и `days` — 400 VALIDATION_FAILED. `days: []` допустимо — сотрудник без рабочих дней. Невалидный UUID `scheduleTemplateId` — 400 VALIDATION_FAILED.

### 8.6. Кастомный валидатор для времени

- Формат: `HH:mm` (регулярка `^\d{1,2}:\d{2}$`; допустимы `"9:00"` и `"09:00"`)
- Парсинг: часы 0–23, минуты 0–59
- Минуты кратны 5
- Для дня: `startTime` < `endTime` (равенство — 400 VALIDATION_FAILED)
- Для перерыва: внутри рабочего дня; перерывы не пересекаются

## 9. Безопасность и доступ

- **Guard:** AdminAuthGuard
- **Проверка:** `requireBusinessMember(adminUserId, businessId)` для всех эндпоинтов
- **Невалидный/истёкший JWT:** 401 `UNAUTHORIZED`
- **Нет доступа:** 404 `NOT_FOUND`
- **Для schedule в Employee API:** проверка существования и принадлежности scheduleTemplateId бизнесу — в одном запросе; при нарушении — 404 NOT_FOUND (без раскрытия причины)
- **Race condition:** при PATCH employee с scheduleTemplateId, если шаблон удалён параллельно — P2025 обрабатывается HttpExceptionFilter → 404 NOT_FOUND
