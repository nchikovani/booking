- `apps/api/src/modules/employee/employee.module.ts`
- Создать `EmployeeService`:

  - `findById(id, businessId)` — проверка принадлежности, 404 при отсутствии
  - `create(businessId, dto)` — дедупликация serviceIds; валидация через repository.validateServiceIdsBelongToBusiness; транзакция: create employee + syncEmployeeServiceLinks; при невалидном — 404
  - `update(id, businessId, dto)` — при отсутствии serviceIds в dto — не менять связи; при наличии — дедупликация, валидация, syncEmployeeServiceLinks в транзакции; при пустом массиве — удалить все связи
  - `delete(id, businessId)` — проверка принадлежности; удаление файла из MinIO (идемпотентно); delete employee (каскад EmployeeService)
  - `uploadPhoto(employeeId, businessId, buffer)` — проверка принадлежности; если employee.imagePath не null — StorageService.delete(employee.imagePath) перед загрузкой; StorageService.upload → employees/{id}/photo.webp; update imagePath
  - `deletePhoto(employeeId, businessId)` — проверка принадлежности; StorageService.delete (идемпотентно); update imagePath = null
  - `syncServiceEmployeeLinks(serviceId, employeeIds, businessId)` — дедупликация; валидация employeeIds через findByIdsAndBusiness; транзакция sync; при невалидном — 404; для вызова из ServiceService
  - `toEmployeeResponse(employee)` — маппинг в response; `imageUrl: employee.imagePath ? this.storageService.getPublicUrl(employee.imagePath) : null`
- Создать `EmployeeModule`: imports PrismaModule, StorageModule; exports EmployeeService
- Валидация serviceIds — в EmployeeRepository (через PrismaClient, без импорта ServiceModule): `validateServiceIdsBelongToBusiness(ids, businessId)`

## Риски и открытые вопросы

- **Порядок шагов:** Сначала Employee (шаги 1–4), затем расширение Service (шаг 5), т.к. ServiceService нуждается в EmployeeService для sync.
- **ParseUUIDPipe:** Проверить, что NestJS ValidationPipe не конфликтует с ParseUUIDPipe для path params.
- **EmployeeService.syncServiceEmployeeLinks:** Метод может жить в EmployeeService (принимает serviceId, employeeIds) или в отдельном EmployeeServiceLinkService. Рекомендация: в EmployeeService — логика симметрична syncEmployeeServiceLinks.
- **ServiceModule импортирует EmployeeModule:** Циклической зависимости не будет — EmployeeModule не импортирует ServiceModule. ServiceModule импортирует EmployeeModule для вызова sync.
