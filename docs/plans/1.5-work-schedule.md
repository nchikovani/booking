# План реализации: Feature 1.5 — Расписание: график работы, перерывы

**Спецификация:** [docs/features/1.5-work-schedule.md](../features/1.5-work-schedule.md)  
**Изменения в 1.4:** [docs/features/1.4-employees.md](../features/1.4-employees.md) — поле `schedule` в Employee API  
**Область:** Backend, БД. Frontend — не входит.

---

## Обзор

Реализация включает:

1. **Schedule Templates** — CRUD шаблонов графика (GET/POST/GET/:id/PATCH/DELETE)
2. **Employee Schedule** — расширение Employee API: поле `schedule` в DTO и ответах, синхронизация графика при create/update

### Шаг 9: Расширение Employee — сервис и репозиторий

**Файлы/компоненты:**

- `apps/api/src/modules/employee/employee.service.ts`
- `apps/api/src/modules/employee/employee.repository.ts`
- `apps/api/src/modules/employee/dto/employee-response.dto.ts`

- [x] **Транзакция create/update:** `EmployeeService.create` и `update` — обернуть в `prisma.$transaction`. Внутри: (1) при scheduleTemplateId — `employeeScheduleService.validateScheduleTemplateId`; (2) валидация serviceIds; (3) create/update employee + syncEmployeeServiceLinks; (4) при schedule — `employeeScheduleService.syncSchedule`. Передавать `tx` в методы, участвующие в транзакции. Атомарность устраняет race (удаление шаблона между проверкой и созданием)
- [x] `EmployeeRepository`: методы `createWithServiceLinks`, `update`, `syncEmployeeServiceLinks`, `validateServiceIdsBelongToBusiness` — добавить опциональный параметр `tx?: Prisma.TransactionClient` для участия в транзакции
- [x] В `EmployeeService.create`: при наличии schedule — вызов syncSchedule внутри транзакции с передачей tx
- [x] В `EmployeeService.update`: при наличии dto.schedule — syncSchedule внутри транзакции; при отсутствии поля — не менять график
- [x] В `EmployeeRepository.findById`, `findByIdAndBusiness`, `findByBusinessId` — include `employeeSchedule` с вложенными days/breaks или scheduleTemplate.days
- [x] В `toEmployeeResponse` добавить `schedule`: вызов `employeeScheduleService.toScheduleResponse(employee.employeeSchedule)` или null
- [x] Добавить `EmployeeScheduleModule` в imports `EmployeeModule`; инжектировать `@Inject(PRISMA) prisma` в EmployeeService для `prisma.$transaction`
- [x] **P2025 (race condition):** при PATCH employee с scheduleTemplateId, если шаблон удалён параллельно — P2025 обрабатывается HttpExceptionFilter → 404 NOT_FOUND. Дополнительная обработка в сервисе не требуется
