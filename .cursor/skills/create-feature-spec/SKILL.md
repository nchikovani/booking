# Создание спецификации фичи

## Когда применять

Применяй этот skill, когда пользователь:
- Просит создать спецификацию/документ фичи
- Описывает новую фичу и хочет получить файл по шаблону
- Говорит «создай фичу X», «create feature spec for X», «новая фича: X»

## Алгоритм

1. **Прочитай шаблон** — `docs/features/_template.md`
2. **Определи номер фичи** — по описанию пользователя или по следующему свободному номеру в `docs/features/` (формат `X.Y`, например 1.3, 1.4)
3. **Сформируй имя файла** — `{X.Y}-{slug}.md`, где slug — латиница, дефисы (например `1.3-services.md`)
4. **Заполни шаблон** на основе описания пользователя
5. **Сохрани файл** в `docs/features/{X.Y}-{slug}.md`
6. **Обнови ссылки** — в «Связанные документы» укажи актуальные Feature X.X

## Формат файла

- Путь: `docs/features/{номер}-{slug}.md`
- Кодировка: UTF-8
- Язык: русский (документация по правилам проекта)

## Примеры триггеров

- «Создай спецификацию фичи 1.3 — категории и услуги»
- «Create feature spec for employee management»
- «Новая фича: расписание работы сотрудников (1.5)»
